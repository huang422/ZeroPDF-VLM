# Data Model: VLM-Assisted ROI Content Detection with Image Preprocessing

**Feature**: 004-vlm-roi-preprocessing
**Created**: 2025-12-31

## Overview

This document defines the data structures, their relationships, and validation rules for the ROI preprocessing pipeline. The model extends existing VLM recognition structures with preprocessing-specific entities.

## Core Entities

### 1. BlankTemplateROI

Represents a blank template ROI image used as reference baseline for template difference.

**Attributes**:
- `template_id` (str): Template identifier (e.g., 'contractor_1', 'contractor_2', 'enterprise_1')
- `field_id` (str): Field identifier (e.g., 'company1', 'signature', 'stamp1')
- `roi_image` (np.ndarray): Blank template ROI image in BGR format, shape (H, W, 3), dtype uint8
- `image_path` (Path): File path to saved blank ROI image: data/{template_id}/blank_rois/{field_id}.png
- `dimensions` (tuple[int, int]): ROI dimensions as (height, width) in pixels
- `extraction_timestamp` (datetime): When the blank ROI was extracted from template

**Validation Rules**:
- `template_id` must match one of the configured templates
- `field_id` must exist in the template's field configuration
- `roi_image` must be valid BGR image with shape (H, W, 3)
- `image_path` must be valid file path and image file must exist
- `dimensions` must match actual roi_image.shape[0:2]

**Relationships**:
- Belongs to one Template (template_id)
- Referenced by BlankTemplateROICache for runtime access
- Used by ROIPreprocessor for template difference

**Storage**:
- File: data/{template_id}/blank_rois/{field_id}.png (PNG image)
- Generated by: update_configs.py during template configuration
- Loaded by: BlankTemplateROICache during pipeline initialization

---

### 2. PreprocessingConfig

Configuration parameters for the 6-step preprocessing pipeline.

**Attributes**:
- `DIFFERENCE_THRESHOLD` (int): Threshold for template difference binarization, range [10, 50], default 25
- `INK_RATIO_THRESHOLD` (float): Minimum ink ratio to consider content present, range [0.001, 0.02], default 0.005 (0.5%)
- `COMPONENT_COUNT_THRESHOLD` (int): Minimum connected components for content, range [1, 10], default 3
- `MIN_BLOB_AREA` (int): Minimum pixel area for valid component, range [5, 100], default 20
- `MORPHOLOGY_LINE_RATIO` (float): Ratio of ROI dimension for line removal kernels, range [0.2, 0.5], default 0.3

**Validation Rules**:
- All thresholds must be within specified ranges
- DIFFERENCE_THRESHOLD must be integer
- INK_RATIO_THRESHOLD must be positive float < 1.0
- COMPONENT_COUNT_THRESHOLD must be positive integer
- MIN_BLOB_AREA must be positive integer
- MORPHOLOGY_LINE_RATIO must be float in (0.0, 1.0)

**Relationships**:
- Singleton configuration loaded from config.py
- Used by ROIPreprocessor for all preprocessing operations

**Storage**:
- File: vlm_pdf_recognizer/recognition/config.py (Python module constants)
- Loaded at: Pipeline initialization time
- Modified by: User editing config.py and restarting application

---

### 3. PreprocessingResult

Output of preprocessing pipeline for a single ROI field.

**Attributes**:
- `field_id` (str): Field identifier matching the ROI
- `has_content` (bool | None): True if content detected, False if empty, None if preprocessing failed/skipped
- `ink_ratio` (float | None): Ratio of non-zero pixels to total pixels, range [0.0, 1.0], None if failed
- `component_count` (int | None): Number of connected components with area > MIN_BLOB_AREA, None if failed
- `processing_time_ms` (float): Time taken for preprocessing pipeline in milliseconds
- `error_message` (str | None): Error description if preprocessing failed, None if successful
- `intermediate_images` (dict[str, np.ndarray] | None): Dict of step name -> image for debugging, None if not saved

**Validation Rules**:
- `has_content` must be bool or None (None only if error occurred)
- `ink_ratio` must be in range [0.0, 1.0] when not None
- `component_count` must be non-negative integer when not None
- `processing_time_ms` must be non-negative float
- If `has_content` is None, `error_message` must not be None
- If `has_content` is bool, `ink_ratio` and `component_count` must not be None

**Relationships**:
- Generated by ROIPreprocessor for each document ROI
- Embedded in RecognitionResult for integration with VLM results
- Used by DocumentRecognitionOutput for validation logic

**Storage**:
- Memory: Part of RecognitionResult dataclass
- File (metadata): output/processed_rois/{document_name}/{field_id}/metadata.json
- File (images): output/processed_rois/{document_name}/{field_id}/0X_*.png

---

### 4. IntermediateImage

Represents one step of the preprocessing pipeline for debugging.

**Attributes**:
- `step_number` (int): Step index (1-6)
- `step_name` (str): Descriptive name ('saturation', 'difference', 'hline_removed', 'vline_removed', 'noise_removed', 'final_binary')
- `image` (np.ndarray): Processed image at this step, grayscale or BGR depending on step
- `file_path` (Path): Path where image is saved: output/processed_rois/{document_name}/{field_id}/0{step_number}_{step_name}.png

**Validation Rules**:
- `step_number` must be in range [1, 6]
- `step_name` must match predefined step names
- `image` must be valid NumPy array
- `file_path` must be valid path in processed_rois directory

**Relationships**:
- Generated by ROIPreprocessor during pipeline execution
- Collected in PreprocessingResult.intermediate_images
- Saved to disk for debugging/threshold tuning

**Storage**:
- File: output/processed_rois/{document_name}/{field_id}/0{step_number}_{step_name}.png

---

### 5. BlankTemplateROICache

Runtime cache for accessing blank template ROI images during document processing.

**Attributes**:
- `cache` (dict[str, dict[str, BlankTemplateROI]]): Nested dict: template_id -> field_id -> BlankTemplateROI
- `templates_dir` (Path): Path to data directory containing template subdirectories
- `loaded_count` (int): Total number of blank ROIs loaded across all templates

**Methods**:
- `load_blank_rois(template_id: str) -> dict[str, BlankTemplateROI]`: Load all blank ROIs for a template
- `get_blank_roi(template_id: str, field_id: str) -> BlankTemplateROI | None`: Get specific blank ROI
- `has_blank_roi(template_id: str, field_id: str) -> bool`: Check if blank ROI exists
- `get_loaded_count() -> int`: Return total number of loaded blank ROIs

**Validation Rules**:
- `templates_dir` must be valid directory path
- All cached BlankTemplateROI objects must pass their own validation
- Template IDs in cache must match configured templates

**Relationships**:
- Loads BlankTemplateROI objects from disk
- Used by ROIPreprocessor to retrieve blank ROIs for comparison
- Initialized by DocumentProcessor at startup

**Storage**:
- Memory: In-memory cache during application runtime
- Source: data/{template_id}/blank_rois/{field_id}.png files

---

### 6. ROIPreprocessor (Component, not data entity)

Executes the 6-step preprocessing pipeline on document ROIs.

**Inputs**:
- `doc_roi_image` (np.ndarray): Document ROI image to analyze
- `blank_roi_image` (np.ndarray): Corresponding blank template ROI
- `config` (PreprocessingConfig): Preprocessing threshold parameters
- `field_id` (str): Field identifier for logging and debug output
- `output_dir` (Path | None): Directory to save intermediate images (None = don't save)

**Outputs**:
- `PreprocessingResult`: Result object with has_content determination and metrics

**Processing Steps**:
1. Step 1 (HSV Preprocessing): Convert to HSV, extract saturation channel
2. Step 2 (Template Difference): Compute absolute difference, apply threshold
3. Step 3 (Structural Filtering): Remove horizontal and vertical lines
4. Step 4 (Noise Removal): Remove salt-and-pepper noise and small blobs
5. Step 5 (Feature Extraction): Calculate ink_ratio and component_count
6. Step 6 (Decision Logic): Determine has_content based on thresholds

## Extended Entities (Modifications to Existing)

### RecognitionResult (MODIFIED)

Existing VLM recognition result extended with preprocessing fields.

**New Attributes**:
- `preprocessing_has_content` (bool | None): Preprocessing pipeline has_content result (None if skipped/failed)
- `preprocessing_ink_ratio` (float | None): Ink ratio metric from preprocessing
- `preprocessing_component_count` (int | None): Connected components count from preprocessing
- `preprocessing_time_ms` (float | None): Preprocessing execution time

**Existing Attributes** (unchanged):
- `field_id` (str)
- `has_content` (bool | None): VLM has_content result
- `content_text` (str | None): VLM extracted text
- `raw_response` (str): VLM raw output
- `parse_success` (bool): VLM JSON parsing status
- `inference_time_ms` (float): VLM inference time
- `retry_count` (int): VLM retry attempts
- `auxiliary_has_content` (bool | None): SIFT-based auxiliary result (deprecated by this feature)
- `auxiliary_similarity_score` (float | None): SIFT similarity score
- `auxiliary_comparison_time_ms` (float | None): SIFT comparison time

**Validation Rules**:
- Existing validation rules preserved
- `preprocessing_has_content` is None for title fields and checkbox fields
- If `preprocessing_has_content` is bool, preprocessing metrics should not be None
- If `preprocessing_has_content` is None (error/skip), preprocessing metrics may be None

---

### DocumentRecognitionOutput (MODIFIED)

Existing VLM document output extended with preprocessing-based validation.

**Modified Methods**:
- `calculate_results_status()`: Uses `preprocessing_has_content` as primary input instead of `auxiliary_has_content`

**Validation Logic Changes**:
- VX1 priority: Use VX1.has_content (heuristic result, not preprocessing)
- Date fields OR: Use preprocessing_has_content for year/month/date fields (fallback to VLM if preprocessing is None)
- Other fields AND: Use preprocessing_has_content for all non-title, non-date fields (fallback to VLM if preprocessing is None)

**Existing Attributes** (unchanged):
- `document_name` (str)
- `page_number` (int)
- `template_id` (str)
- `field_results` (List[RecognitionResult])
- `results` (bool)
- `processing_timestamp` (datetime)
- `total_processing_time_ms` (float)

## Data Flow

```text
1. Initialization Phase (update_configs.py):
   Template Image → ROI Extraction → BlankTemplateROI → Save to disk (data/{template_id}/blank_rois/)

2. Pipeline Startup (main.py, pipeline.py):
   Disk (blank_rois/) → BlankTemplateROICache.load_blank_rois() → In-memory cache

3. Document Processing (vlm_recognizer.py):
   Document ROI → ROIPreprocessor.preprocess(doc_roi, blank_roi, config) → PreprocessingResult

4. VLM Integration:
   PreprocessingResult → Embed in RecognitionResult.preprocessing_* fields

5. Validation:
   RecognitionResult.preprocessing_has_content → DocumentRecognitionOutput.calculate_results_status() → results (bool)

6. Output Export (output.py):
   RecognitionResult → JSON/CSV with preprocessing columns
   IntermediateImages → Saved to output/processed_rois/
```

---

## State Transitions

### BlankTemplateROI Lifecycle
1. **Generated**: Created by update_configs.py from template image
2. **Saved**: Written to data/{template_id}/blank_rois/{field_id}.png
3. **Loaded**: Read into BlankTemplateROICache at pipeline startup
4. **Cached**: Kept in memory for document processing
5. **Referenced**: Used by ROIPreprocessor for template difference
6. **Stale**: Becomes outdated if template ROI coordinates change (requires regeneration)

### PreprocessingResult Lifecycle
1. **Created**: ROIPreprocessor executes 6-step pipeline
2. **Populated**: has_content, metrics, and optional intermediate images filled
3. **Embedded**: Attached to RecognitionResult object
4. **Used for Validation**: preprocessing_has_content drives results calculation
5. **Exported**: Written to JSON/CSV output files
6. **Discarded**: Memory freed after document processing completes

## Validation Matrix

| Entity | Validation Rule | Error Handling |
|--------|----------------|----------------|
| BlankTemplateROI | Image file exists at expected path | Log warning, skip preprocessing for that field |
| BlankTemplateROI | Image dimensions match config | Resize document ROI to match, log warning |
| PreprocessingConfig | All thresholds in valid ranges | Raise ValueError at config load time |
| PreprocessingResult | has_content is None only if error occurred | Set error_message, continue with VLM |
| PreprocessingResult | Metrics populated when has_content is bool | Assert in validate() method |
| RecognitionResult | preprocessing_has_content None for title/checkbox | Skip preprocessing for these field types |
| DocumentRecognitionOutput | All preprocessing results valid | Fallback to VLM has_content if preprocessing failed |

## Performance Considerations

### Memory Usage
- **BlankTemplateROICache**: ~3 templates × 13 ROIs × ~50KB per ROI = ~2MB total
- **IntermediateImages**: 6 images × ~100KB per image × 13 ROIs = ~8MB per document (only if debug enabled)
- **PreprocessingResult**: Negligible (~1KB per ROI including metrics)

### Disk Usage
- **Blank ROIs**: ~3 templates × 13 ROIs × ~50KB = ~2MB total
- **Debug Images**: ~8MB per processed document (can accumulate rapidly in batch processing)
- **Recommendation**: Periodically clean output/processed_rois/ directory

### Processing Time
- **BlankTemplateROI loading**: ~10ms per template (one-time at startup)
- **Preprocessing per ROI**: Target <100ms (6 steps × ~15ms average)
- **Debug image saving**: ~20-30ms per ROI (6 PNG writes)
- **Total overhead**: ~10-20% increase to existing pipeline time
